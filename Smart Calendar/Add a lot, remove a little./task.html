Description

Now you can not only read data but also handle errors. Fine! But using the program, adding one note at a time, closing it, and restarting it, is extremely inconvenient! What if the user wants to add several notes at once, or delete some notes? Let's supplement your program by teaching it to ask the user how many notes they want to add or delete.

Please note that the events in the files are removed before each test run for the tests to work correctly.

Objectives

You add new functions:

  1. Display all current notes after displaying the current date and time. If there are no current notes, skip this step.
  2. Ask the user how many notes they want to add or delete. Then you read the number. A positive number means that the user wants to add several notes, and negative means that the user decides to delete that many notes.
  3. To add notes, read the data and the text and save the result in a file.
  4. To delete a note, ask the user to enter the note's text and remove information from your file. If such a note is not found, you need to print an error message and ask again
  5. In the end, print all saved notes

Current date and time:

Enter date and time of note #2 (in format «YYYY-MM-DD HH:MM»):
