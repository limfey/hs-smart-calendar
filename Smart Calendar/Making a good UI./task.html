Description

The main functionality of the calendar is ready. However, with a large number of entries, this program looks cumbersome and awkward. It lacks a competent user interface! Therefore, at the final stage of the project, this interface must be implemented.

Let's define the initial behavior of the program: Let it display not only the current date and time but also all the notes, as well as the dates of birth corresponding to the present day. Next, let the program wait for user input.

Please note that the events in the files are removed before each test run for the tests to work correctly.

Objectives

In this task, your program should print the current date and time as before, print a message Enter the command (add, view, delete, exit) and get input. if the user wrote a command that is not in the proposed menu, then you need to write This command is not in the menu and print the start message again.

If the user writes add, the program should ask what to add by a message What do you want to add (note, birthday)?. Next, the subroutine for adding a record is started.

  • If the user chooses note, your program should print How many notes do you want to add? and add as many notes to the file as the user specified, before.
  • If the user choosesbirthday, your program should print How many dates of birth do you want to add? and add as many dates of birth to the file as the user specified, before.

If the user writes view, the program should print a message What do you want to view (date, note, name)? and get a command:

  • date: print Enter date (in format «YYYY-MM-DD»):, get a date, print Found (number of notes) note(s) and (number of notes) date(s) of birth on this date: and print the corresponding notes, dates of birth, and the number of days before these events as before.
  • name: display the person's date of birth by name as before. If such a person is not found, print No such person found. Try again: and get the text again.
  • note: your program should print Enter text of note: and display a note if a user-written substring is found in a note (for example, View note Doctor should find all notes where the text contains Doctor's substring). If such text is not found, print No such note found. Try again: and get the text again.

If a user writes delete, the program should ask What do you want to delete (date, note, name)?, get input, and show the found notes or the dates of birth as in the view command. Then your program should ask for confirmation before deletion of every single note or date of birth: Are you sure you want to delete and show the corresponding note or the name of the person. If the user writes yes, your program should delete the found information. If the user writes no, your program print Deletion canceled. After deleting the note or the date of birth, your program should print the message Note deleted! or Birthdate deleted! correspondingly.

After completing the actions, your program should return to the main menu and repeat the entire algorithm again. If the user writes exit, then the program should finish receiving commands and executing them.
