Description

Your program is getting better and better! But you may face a common programming problem — the user enters the data in an incorrect format!

Let's take a simple example: a user wants to enter a note in the 13th month. But there is no month with this number! What will happen then? Your program (and any bad program) will likely crash with an exception. But you are doing a good one! To do this, ask the user to enter the date and time with one input, and try to handle the exceptions associated with this input.

Please note that the events in the files are removed before each test run for the tests to work correctly.

Objectives

In this stage, you need to change the user interface for your program.

  1. Display the current date and time (in hours and minutes);
  2. Ask for the date and time of the event in the one-line format: (YYYY-MM-DD HH:MM);
  3. If the user makes a mistake, print the corresponding message and ask the user for input again:
    • Print Incorrect format. Please try again (use the format «YYYY-MM-DD HH:MM») if you can't parse the data input by the user;
    • Print Incorrect month value. The month should be in 01-12. if the user inputs a month out of range;
    • Print Incorrect hour value. The hour should be in 00-23. if the user inputs an hour out of range;
    • Print Incorrect minute value. The minutes should be in 00-59. if the user inputs a minute value out of range.
  4. When the input is correct, ask for the text of the note.

Then the program should display how much time is left until this note in the format:
Before the event note "Text" remains: XX day(s), HH hour(s) and MM minute(s).

Examples

Example 1: the user inputs the correct data

Current date and time:
2020-05-13 20:33

Let's add a note to the calendar.
Enter date and time (in format «YYYY-MM-DD HH:MM»):
> 2020-06-20 18:00
Enter a text:
> Visit a doctor

Before the event note "Visit a doctor" time remains: 37 day(s), 21 hour(s) and 26 minute(s).

Example 2: mistakes in the user input

Current date and time:
2020-05-13 20:33

Let's add a note to the calendar.
Enter date and time (in format «YYYY-MM-DD HH:MM»):
> 2020.08,19 19-threezero
Incorrect format. Please try again (use the format «YYYY-MM-DD HH:MM»):
> 2020-13-19 20:00
Incorrect month value. The month should be in 1-12.
Please try again:
> 2020-08-19 19:30
Enter a text:
> Go to the gym

Before the event note "Visit a doctor" time remains: 37 day(s), 21 hour(s) and 26 minute(s).
Before the event note "Go to the gym" time remains: 97 day(s), 22 hour(s) and 56 minute(s).
